Red Hat JBoss Developer Studio Early Access

We wish to give customers and users early access to Developer Studio so we can make the next version of Red Hat JBoss Developer Studio an even better environment for developing on the JBoss Enterprise Platform.

We do our best to do this with our developers and internal QA, but we would also like to have real developers and real users try out early builds to make sure we don't just cover the issues we find but also what upcoming users of devstudio might bump into.

Program Structure And Terms

The Red Hat JBoss Developer Studio Early Access program ran as a unmanaged program in which a small number of existing customers may choose to participate. The program started in June 2011 and has now completed. Although these Early Access releases are not officially supported by Red Hat Global Support team, the Red Hat JBoss Developer Studio product team will directly assist with participant questions. At the conclusion of the program, we expect customers to use the GA binaries in their ongoing development process. After the GA release, any kind of support for Early Access releases will cease to exist. Click here for more information.

Program Complete!

  • This program has completed. Red Hat JBoss Developer Studio 3.0 is now available in the CSP (Customer Support Portal). If you are not an existing customer, or do not yet have a Red Hat JBoss Developer Studio subscription, please go to http://www.redhat.com/products/jbossenterprisemiddleware/developer-studio/ to purchase a subscription to the CSP.

  • Future Early Access program releases will be posted here. Watch this page for details.

Feedback

The Red Hat JBoss Developer Studio product team will directly assist with participant questions on a best effort basis. Since these builds are not publically available please contact us via this mailing list devstudio-earlyaccess-list@redhat.com.

We will then evaluate the feedback and if needed, we will report the issue in JIRA, then provide you a link to where the discussion can continue.

Thank you!
